# Context Docs

Core startup context is defined by the Context Loading Contract in `AGENTS.md`. Keep these files compact and avoid duplicating instructions from workflows or skills.

- `project-overview.md`: repository purpose and structure.
- `coding-standards.md`: shared rule IDs plus code, test, and commit rules for target apps.
- `design.md`: core product-wide frontend visual contract for user-facing UI work.
- `design-extended.md`: extended density, component-default, spacing, and motion guidance loaded only when needed.
- `visual-acceptance.md`: acceptance checklist and reporting contract for user-facing UI work.
- `current-feature.md`: active spec, optional epic sub-spec queue, and completed history.

Do not load `specs/` unless an active or named spec is needed.
